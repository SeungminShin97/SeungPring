# 공통 Javadoc 템플릿 (SeungSpring 프로젝트 기준)

> 목적
> - **구조적 경계 명확화**
> - **흐름 중심 설명**
> - **래퍼런스 표시**
>
> 이 템플릿은 Spring / Tomcat 구조를 참고하되,  
> **직접 구현한 프레임워크의 설계 의도를 설명하기 위한 Javadoc 표준**이다.

---

## 1. 인터페이스용 Javadoc 템플릿 (계약 · 경계 중심)

```java
/**
 * [역할 요약 한 줄]
 *
 * <p>
 * 이 인터페이스는 [A 영역]과 [B 영역] 사이의
 * 책임 경계를 명확히 분리하기 위해 정의됩니다.
 * </p>
 *
 * <p>
 * 주로 다음과 같은 역할을 담당합니다.
 * <ul>
 *     <li>[역할 1]</li>
 *     <li>[역할 2]</li>
 *     <li>[역할 3]</li>
 * </ul>
 * </p>
 *
 * @see 관련_인터페이스_또는_구현체
 * @see <a href="참고 URL">참고 문서 또는 레퍼런스</a>
 */
public interface XxxInterface {
}
```

## 2. 메서드용 Javadoc 템플릿 (요청 흐름 기준)
```java
/**
 * [이 메서드가 수행하는 핵심 역할 한 줄]
 *
 * <p>
 * 이 메서드는 전체 처리 흐름 중
 * [정확한 단계 / 경계 지점]에 해당합니다.
 * </p>
 *
 * <p>
 * 처리 결과는 [반영 대상 객체 / 소비 방식]에 기록되며,
 * 반환값의 유무와 관계없이
 * 후속 단계에서 해당 결과를 기반으로 처리가 이어집니다.
 * </p>
 *
 * @param xxx [파라미터 의미 및 책임]
 * @param yyy [파라미터 의미 및 책임]
 * @return 반환값이 존재하는 경우에만 명시
 * @throws XxxException [예외 발생 조건]
 */
 ```

## 3. 추상 클래스용 Javadoc 템플릿 (확장 포인트 명시)
```java
/**
 * [공통 처리 흐름 또는 알고리즘을 제공하는 추상 클래스 설명]
 *
 * <p>
 * 이 클래스는 [공통 동작]을 기본 구현으로 제공하며,
 * 세부 동작은 하위 클래스에서 확장하도록 설계되었습니다.
 * </p>
 *
 * <p>
 * 하위 구현체는 다음 메서드를 반드시 구현해야 합니다.
 * <ul>
 *     <li>{@link #abstractMethod()}</li>
 * </ul>
 * </p>
 *
 * <p>
 * 본 클래스는 [상태 보유 여부],
 * [스레드 안전성 정책]에 대한 제약을 가집니다.
 * </p>
 */
public abstract class XxxAbstract {
}
```

## 4. 구현 클래스용 Javadoc 템플릿 (의존성 · 선택 이유 명시)
```java
/**
 * {@link XxxInterface}의 기본 구현체입니다.
 *
 * <p>
 * 이 구현체는 [구체적인 전략 또는 방식]을 사용하여
 * [처리 대상 책임]을 수행합니다.
 * </p>
 *
 * <p>
 * 내부적으로 다음 컴포넌트에 의존합니다.
 * <ul>
 *     <li>{@link Aaa} - [역할]</li>
 *     <li>{@link Bbb} - [역할]</li>
 * </ul>
 * </p>
 *
 * <p>
 * 본 구현은 [이 방식을 선택한 이유]를 기준으로 설계되었으며,
 * [의도적으로 제외한 대안 또는 패턴]은 사용하지 않습니다.
 * </p>
 */
public class DefaultXxx implements XxxInterface {
}
```

## 5. 팩토리 클래스용 Javadoc 템플릿 (선택 정책 명확화)
```java
/**
 * [선택 기준]에 따라
 * 적절한 {@link Xxx} 구현체를 반환하는 팩토리 클래스입니다.
 *
 * <p>
 * 이 클래스는 <strong>싱글톤(Singleton)</strong>으로 제공되며,
 * 애플리케이션 전반에서 동일한 선택 정책을 유지합니다.
 * </p>
 *
 * <p>
 * 새로운 구현체가 추가될 경우,
 * 이 클래스의 선택 로직을 확장해야 합니다.
 * </p>
 *
 * <p>
 * 지원하지 않는 입력이 전달될 경우
 * {@link IllegalArgumentException}이 발생합니다.
 * </p>
 */
public class XxxFactory {
}
```

## 6. 문서화 원칙 (필수)
- 경계가 어디인지 반드시 명시
- 흐름 상 위치를 항상 설명
- “무엇을 한다”보다 “왜 여기까지만 하는가”를 설명
- 래퍼런스가 있는 경우 반드시 명시
